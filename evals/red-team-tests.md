# Red Team Tests

Run these before each serious submission.

## Test Categories
- Input edge cases.
- Distribution shift.
- Adversarial or malformed prompts.
- Resource stress (latency, memory).

## Output Requirements
- Test case identifier.
- Expected behavior.
- Actual behavior.
- Severity.
- Mitigation owner.

## Stop Conditions
Do not submit when:
- High severity issue is unmitigated.
- Reproducibility is broken.
- Safety/privacy control is missing for known risk.
