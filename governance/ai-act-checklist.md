# AI Act Checklist (Practical Team Version)

Complete per challenge before final submission.

## System Definition
- [ ] Purpose of the AI system is clearly documented.
- [ ] Intended users and use context are documented.

## Risk and Control
- [ ] Key risks identified and logged in `risk-register.md`.
- [ ] Controls are assigned to owners.
- [ ] Residual risk is accepted explicitly.

## Data Governance
- [ ] Data sources are listed.
- [ ] Data rights/usage basis is documented.
- [ ] Sensitive data handling controls are defined.

## Transparency and Traceability
- [ ] Model card created.
- [ ] Data card created.
- [ ] Decision log updated.

## Security and Robustness
- [ ] Abuse and edge-case tests executed.
- [ ] High-severity failures mitigated or accepted explicitly.

## Human Oversight
- [ ] Human owner assigned for go/no-go.
- [ ] Escalation path is documented.
