A Python library repository that implements various multi-agent path finding algorithms. It may not be published on PyPI but can be cloned and used directly in projects.

import asyncio, json, subprocess, websockets, time

class PlannerProc:
    def __init__(self, cmd):
        self.p = subprocess.Popen(
            cmd, stdin=subprocess.PIPE, stdout=subprocess.PIPE, text=True, bufsize=1
        )

    def decide(self, state_json: str) -> str:
        # send one line JSON, read one line JSON
        self.p.stdin.write(state_json.replace("\n", " ") + "\n")
        self.p.stdin.flush()
        return self.p.stdout.readline()

async def play():
    planner = PlannerProc(["./planner_binary"])  # or ["python","planner.py"] etc.
    async with websockets.connect("wss://game.ainm.no/ws?token=YOUR_TOKEN") as ws:
        while True:
            msg = await ws.recv()
            state = json.loads(msg)
            if state.get("type") == "game_over":
                return

            t0 = time.time()
            actions_msg = planner.decide(json.dumps(state))
            # Optionally enforce time budget:
            if time.time() - t0 > 1.9:
                # fallback if planner too slow
                actions_msg = json.dumps({"actions": []})

            await ws.send(actions_msg)

asyncio.run(play())

Each item is picked from Manhattan distance 1, so for every item i define its pickup neighborhood:
•	Adj(i) = {walkable cells c | manhattan(c, item_cell) == 1}
Treat “picking item i” as “visit one of the cells in Adj(i) and execute PICK”.
Same for dropoff: define Adj(dropoff).

•	Decision variables:
o	which bot picks each required item
o	which adjacent cell it will pick from (choice in Adj(i))
o	the order of picks for each bot
o	when each bot returns to dropoff (capacity constraint)
•	Objective:
o	maximize completed orders within 300 ticks
o	tie-breaker: minimize total travel distance, minimize “late dropoff”

•	optionally 1–2 items from the preview order (if it’s beneficial and won’t delay current completion)

•	congestion_penalty(route)
•	spawn_separation_penalty(tick 0..k) (see below)

•	finish the current order ASAP,
•	and only pre-pick preview items if they don’t meaningfully delay that.

score = + BIG * (#orders_completed_by_T)
- α * (time_to_finish_current_order)
- β * (total_travel_distance)
- γ * (predicted_conflicts / congestion)

•	If the order is not completed, preview items stay stuck in inventory, and the bot must vacate the drop-off anyway.
